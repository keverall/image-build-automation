# definition

## Description

Standalone script executed by GitLab CI/CD pipelines to perform maintenance mode operations. Wraps Set-MaintenanceMode with GitLab-specific context, logging, and callback support. Outputs results as JSON artifacts and sends web callbacks on completion. Integrates with GitLab pipeline variables for configuration.

## Parameters

| Parameter | Description |
|-----------|-------------|
| `-ACTION` | Maintenance action: enable, disable, or validate (required) |
| `-CLUSTER_ID` | Target cluster identifier (required) |
| `-MODE` | Maintenance mode type: scom or oneview (default: 'scom') |
| `-START` | Maintenance window start time (ISO 8601 format) |
| `-END` | Maintenance window end time (ISO 8601 format) |
| `-CONFIG_DIR` | Configuration directory path (default: 'configs') |
| `-DRY_RUN` | Validate without executing changes |
| `-NO_SCHEDULE` | Skip scheduling future maintenance window |
| `-GITLAB_TOKEN` | GitLab job token (from GITLAB_JOB_TOKEN environment variable) |
| `-CI_PROJECT_ID` | GitLab project ID (from CI_PROJECT_ID environment variable) |
| `-CI_PIPELINE_ID` | Pipeline ID (from CI_PIPELINE_ID environment variable) |
| `-CI_JOB_ID` | Job ID (from CI_JOB_ID environment variable) |
| `-CALLBACK_URL` | URL for completion callback (from MAINTENANCE_CALLBACK_URL) |
| `-CALLBACK_API_KEY` | API key for callback authentication (from MAINTENANCE_API_KEY) |

## Examples

### Example 1
```powershell
pwsh -File scripts/gitlab/Invoke-GitLabMaintenance.ps1 -ACTION enable -CLUSTER_ID 'CLUSTER01'
```

### Example 2
```powershell
pwsh -File scripts/gitlab/Invoke-GitLabMaintenance.ps1 -ACTION validate -CLUSTER_ID 'CLUSTER01' -DRY_RUN
```
