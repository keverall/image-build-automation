# for

## Description

Initiates GitLab pipeline via trigger API and optionally waits for completion. Sends web callback with results when pipeline finishes. Can be used two ways: 1. Dot-sourced: Defines Send-GitLabMaintenanceRequest function for reuse 2. Direct execution: Runs immediately with provided parameters Integrates with GitLab trigger tokens, job tokens, and pipeline variables.

## Parameters

| Parameter | Description |
|-----------|-------------|
| `-Action` | Maintenance action: enable, disable, or validate (required when dot-sourcing) |
| `-TargetId` | Target cluster or server identifier (required when dot-sourcing) |
| `-Start` | Maintenance window start time (ISO 8601 format) |
| `-End` | Maintenance window end time (ISO 8601 format) |
| `-ConfigDir` | Configuration directory path (default: 'configs') |
| `-DryRun` | Validate without executing changes |
| `-GitLabUrl` | GitLab instance URL (from GITLAB_URL environment variable) |
| `-ProjectId` | GitLab project ID (from GITLAB_PROJECT_ID environment variable) |
| `-TriggerToken` | GitLab CI trigger token for pipeline initiation (from GITLAB_TRIGGER_TOKEN) |
| `-GitRef` | Git branch/reference to trigger pipeline on (default: 'main') |
| `-CallbackUrl` | URL to send completion notification (from MAINTENANCE_CALLBACK_URL) |
| `-CallbackApiKey` | API key for callback authentication (from MAINTENANCE_API_KEY) |
| `-TimeoutSeconds` | Maximum wait time for pipeline completion in seconds (default: 600) |
| `-JobToken` | GitLab job token for API access (from GITLAB_JOB_TOKEN) |
| `-SkipValidation` | Skip parameter validation when running directly (for testing) |
